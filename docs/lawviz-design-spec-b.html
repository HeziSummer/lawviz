01

设计哲学

方案B的核心命题：法律权威感与时尚知性审美可以共存。律析的核心用户是青年律师，她们同时阅读《Vogue China》与最高院判决书，对视觉的要求与对专业的要求同样苛刻。

三个关键词

知性（Intellectual）
Playfair Display衬线字体

Playfair Display 斜体作为展示字体，给产品带来编辑感与文学气质。对比以体现智识，而非以装饰取悦。专业不等于冷漠。

克制（Restrained）
单点缀色无装饰渐变

深靛紫是唯一点缀色，珊瑚橙仅用于行动按钮。整个色彩系统只有两个非中性色。少即是多——每一处色彩的出现都应当令人印象深刻。

温度（Warmth）
暖象牙底珊瑚 CTA

底色用象牙白而非冷白，给严肃法律产品带来一丝呼吸感。珊瑚色行动按钮传递"邀请感"，而非"命令感"——符合青年律师作为服务提供者的身份认知。

色彩体系

CSS Custom Properties（根变量）

:root {
  /* ── 基础 ── */
  --bg:           oklch(98% 0.005 75);    /* 象牙白页面底色 */
  --surface:      oklch(100% 0 0);         /* 纯白卡片/导航 */
  --surface-2:    oklch(96% 0.007 75);     /* 次级背景、斑马纹 */
  --fg:           oklch(17% 0.02 285);     /* 主文字，深靛近黑 */
  --muted:        oklch(50% 0.016 285);    /* 次级文字、标签 */
  --border:       oklch(88% 0.008 75);     /* 暖调分隔线 */
  --border-cool:  oklch(90% 0.006 285);    /* 冷调分隔线（卡片） */

  /* ── 点缀色：深靛紫（主）── */
  --accent:       oklch(40% 0.17 294);     /* 全量：logo、强调、选中 */
  --accent-light: oklch(40% 0.17 294 / 0.07);  /* 浅底：hover 背景 */
  --accent-mid:   oklch(40% 0.17 294 / 0.18);  /* 中透：描边 */

  /* ── 行动色：珊瑚橙（仅 CTA）── */
  --coral:        oklch(68% 0.15 18);      /* 填充按钮背景 */
  --coral-soft:   oklch(68% 0.15 18 / 0.1);    /* 浅底：辅助场景 */
}

色彩语义分配

变量 视觉值 oklch 用途
--bg 象牙白 98% 0.005 75 页面底色、Hero 背景
--surface 纯白 100% 0 0 导航栏、卡片、弹窗
--fg 深靛近黑 17% 0.02 285 H1–H3、body 正文
--muted 靛灰 50% 0.016 285 次级文字、placeholder、标注
--border 暖米边 88% 0.008 75 分隔线、表格行线
--accent 深靛紫 40% 0.17 294 品牌色、logo、强调文字、激活状态
--coral 珊瑚橙 68% 0.15 18 仅用于主行动按钮（注册/开始使用）
唯一性规则：整个产品中 --coral 只出现在"主行动按钮"（免费体验、立即注册、开始使用）。其他任何元素——标签、徽章、图标、插图、状态——均不使用珊瑚色，以保持其行动含义的稀缺性。

禁止使用的色彩

  • × 任何 oklch(h: 20–70) 暖棕/米橙/杏色的页面背景或大面积色块（非品牌要求）
  • × 紫色/蓝紫渐变作为 Hero 或卡片背景
  • × 红色（错误状态除外）、绿色（成功状态除外）在品牌 UI 中使用
  • × 超过两个非中性色同时出现在同一视区内

加载与状态

  • ✓AI 生成中：全屏居中的"AI 正在生成可视化..."文案 + accent 色点状脉冲动画，背景 var(--bg)
  • ✓追问阶段轮次提示："第 X 轮，最多 3 轮"，居中小字 muted
  • ✓积分不足时：行动按钮禁用 + 错误提示文字（珊瑚色 13px）

导航行为

  • ✓导航栏 sticky + backdrop-filter: blur(12px)，滚动后保持可见
  • ✓已登录用户访问 / 自动重定向至 /dashboard
  • ✓生成页内三阶段切换（填表→追问→生成中）由前端 state 控制，URL 不变
  • ×不使用 scrollIntoView（可能在预览 iframe 内触发布局异常）

10

禁止事项

以下任何一条在代码 Review 中出现，视为阻断性问题，必须在合并前修复。
  • ×渐变背景：不在任何大面积区域使用 CSS 渐变（gradient）。背景色只能是 var(--bg) 或 var(--surface)。
  • ×Emoji 功能图标：不使用 ✨🚀🎯 等 emoji 替代功能图标或装饰。如需图标，使用 Lucide React 线性图标，stroke-width: 1.5，颜色 var(--muted)。
  • ×左侧彩色边框卡片：不使用 border-left: 4px solid <accent> 的装饰性卡片样式。
  • ×Inter / Roboto / Arial 作展示字：展示标题（H1/H2）只用 Playfair Display，不用上述字体。正文系统字体栈不变。
  • ×虚构数据：不在产品页面填写不存在的用户数量、律师证书数、成功率等统计数字。如需占位，用短横线 — 或括号描述 [上线后更新]。
  • ×设计器控件暴露：产品页面不能出现"选择平台"下拉、"方案切换"标签、"生成目标数量"徽章等设计过程控件。
  • ×珊瑚色滥用：var(--coral) 仅限主行动按钮（注册、立即开始使用）。不用于标签、提示、装饰线、图表。
  • ×每个标题旁加图标：H2/H3 标题前不加图标，让 Playfair 斜体字本身承担视觉锚点。
  • ×暖米/杏色/奶油色大背景：var(--bg) 的 oklch H 值已控制为 75（略偏暖），不再进一步加深暖色调。不引入 #FFF8F0 等 AI 风格暖底色。
  • ×scrollIntoView()：预览 iframe 中存在该调用会引发布局错乱，禁止在任何场景使用，改用原生 scroll 方法。
